# vocational_training_SST
# コミュニケーションSST アプリ

Web受託開発におけるコミュニケーションスキルを訓練するアプリケーションです。

## 🎯 概要

AIがクライアント役となり、受託開発でよくあるコミュニケーション場面をシミュレーションできます。

### 訓練できるスキル

1. **要件確認** - クライアントの曖昧な要望から具体的な要件を引き出す
2. **技術と非技術の翻訳** - 技術的な内容を非エンジニアに分かりやすく説明する
3. **曖昧さの構造化** - 抽象的な要望を具体的な仕様に落とし込む
4. **責任範囲の明文化** - 誰が何を担当するか明確にする
5. **合意形成** - クライアントとの合意を文書で確認する

## 🎤 面接練習シミュレーション機能（新機能）

### 概要

Web系受託開発企業の採用面接を想定した面接練習シミュレーション機能です。AIが面接官役となり、実際の面接に近い環境で練習できます。

### 面接の種類

1. **1次面接（人事面接）**
   - 基本的なコミュニケーション能力の確認
   - 志望動機、自己PR
   - 企業文化とのマッチング確認

2. **2次面接（技術面接）**
   - 技術スキルの深掘り
   - プロジェクト経験の詳細確認
   - 技術的な課題解決能力の評価

3. **3次面接（最終面接）**
   - 経営層・役員による面接
   - 企業ビジョンへの共感度確認
   - 長期的なキャリアプランの確認

### 企業文化・風土のカスタマイズ

面接練習開始前に、企業独自の文化・風土を入力できます：

- **企業理念・ビジョン**: 会社が大切にしている価値観
- **開発文化**: アジャイル、ウォーターフォール、リモートワークなど
- **求める人物像**: チームワーク重視、自走力、技術志向など
- **企業の特徴**: 残業文化、フラットな組織、教育制度など

入力された情報を基に、AIが企業の文化に沿った質問や評価を行います。

### シミュレーションの流れ

1. **面接種類の選択**: 1次/2次/3次面接から選択
2. **企業情報の入力**: 企業文化、風土、求める人物像などを入力（任意）
3. **面接開始**: AIが面接官として質問を開始
4. **応答**: 面接官の質問に回答
5. **評価**: 面接終了後、以下の観点から評価
   - 第一印象・コミュニケーション態度
   - 回答の論理性・明確さ
   - 企業文化とのマッチング度
   - 技術的な理解度（技術面接の場合）
   - 熱意・意欲の伝わり方

### 評価基準

#### 1次面接（人事面接）
- **コミュニケーション能力**: 明確で分かりやすい説明ができるか
- **志望動機の明確さ**: なぜこの企業・業界を選んだのか
- **企業文化とのマッチング**: 企業の価値観との適合性
- **基本的なマナー**: 言葉遣い、姿勢、聞く態度

#### 2次面接（技術面接）
- **技術的理解度**: 専門知識の深さと正確性
- **問題解決能力**: 課題に対するアプローチ方法
- **プロジェクト経験**: 具体的な成果と学び
- **技術への探求心**: 最新技術への関心、学習意欲

#### 3次面接（最終面接）
- **ビジョンへの共感**: 企業の将来像への理解と共感
- **キャリアプラン**: 長期的な成長ビジョン
- **リーダーシップ**: チームを牽引する力
- **企業への貢献意欲**: 具体的な貢献イメージ

### データ保存

面接評価結果は以下の形式で保存・エクスポート可能：
- テキスト形式でコピー
- CSV形式でダウンロード

## 🛠️ 技術スタック

- **Frontend**: Next.js 16 (App Router), React 19, TypeScript, Tailwind CSS
- **Backend**: Next.js API Routes
- **Database**: PostgreSQL (NeonDB)
- **ORM**: Drizzle ORM
- **AI**: Google Gemini 2.5 Flash-Lite API

## 🚀 セットアップ

### 1. 環境変数の設定

`.env.example` をコピーして `.env` を作成し、必要な値を設定してください。

```bash
cp .env.example .env
```

```env
# NeonDB Connection String
DATABASE_URL=postgresql://username:password@host/database?sslmode=require

# Gemini API Key
GEMINI_API_KEY=your_gemini_api_key_here
```

### 2. Gemini API キーの取得

1. [Google AI Studio](https://aistudio.google.com/) にアクセス
2. 「Get API key」をクリック
3. 新しいAPIキーを作成
4. `.env` ファイルに設定

### 3. NeonDB のセットアップ

1. [Neon](https://neon.tech/) でアカウントを作成
2. 新しいプロジェクトを作成
3. 接続文字列を `.env` ファイルに設定

### 4. データプライバシーポリシー

**🔒 データの取り扱いについて**

このアプリケーションはGoogle Gemini 2.5 Flash-Lite APIを利用しています。

#### Google Gemini API のデータポリシー

Google の公式ポリシーに基づき、以下が保証されています:

- **学習に使用されません**: Gemini APIを通じて送信されたデータは、Googleのモデル改善やトレーニングには使用されません
- **データのプライバシー**: 会話履歴や評価データは、APIリクエストの処理にのみ使用されます
- **データの所有権**: 入力および出力データの所有権はユーザーに帰属します

#### 本アプリケーションについて

- **無料枠あり**: Gemini APIには無料枠が用意されています（制限あり）
- **API経由の利用**: すべてのAIリクエストはGemini APIに送信されます
- **データの用途**: 会話履歴や評価データは、トレーニング用途として保存・学習されることはありません

参考: [Google AI Studio](https://aistudio.google.com/)

### 5. 依存関係のインストール

```bash
npm install
```

### 6. データベースのセットアップ（オプション）

データベースを使用する場合：

```bash
# スキーマをDBにプッシュ
npm run db:push

# または、マイグレーションファイルを生成して実行
npm run db:generate
npm run db:migrate
```

### 7. 開発サーバーの起動

```bash
npm run dev
```

http://localhost:3000 でアプリにアクセスできます。

## 🎮 使い方

1. **トレーニング開始**: カテゴリと難易度を選択して「トレーニングを開始」
2. **会話**: クライアント（AI）と会話しながら要件を確認
3. **評価**: 3回以上やり取りした後、「会話を終了して評価」をクリック
4. **フィードバック確認**: 5つの評価基準に基づいたスコアと改善点を確認

## 🔧 カスタマイズ

### シナリオの追加

`src/backend/services/ai.ts` の `generateScenario` 関数でプロンプトを調整できます。

### 評価基準の変更

`evaluateSession` 関数のプロンプトを編集することで、評価基準をカスタマイズできます。
